# 京东仓储管理系统 - 手动启动指南

## 一、系统环境要求

| 依赖项 | 版本要求 | 说明 |
|:---|:---:|------|
| Java | 8+ | 运行时环境 |
| Maven | 3.8+ | 项目构建工具 |
| MySQL | 5.7+ | 数据库 |
| Redis | 6.0+ | 缓存服务 |

## 二、启动前准备

### 1. 检查数据库连接
确保 MySQL 服务已启动，数据库配置如下：
- **数据库名**: `wms_db`
- **用户名**: `root`
- **密码**: `123456`
- **端口**: `3306`

### 2. 检查 Redis 配置
确保 Redis 服务可访问：
- **地址**: `localhost`
- **端口**: `6379`
- **密码**: 无

## 三、启动步骤

### 步骤 1：启动 Redis 服务

打开 **命令提示符(cmd)** 或 **PowerShell**，执行以下命令：

```bash
cd D:\毕设\Redis
redis-server.exe redis.conf
```

> **注意**: 保持此窗口打开，关闭窗口会停止 Redis 服务。

### 步骤 2：编译项目

打开 **新的命令提示符**，进入项目目录并编译：

```bash
cd D:\毕设\demo
mvn clean package -DskipTests
```

> **参数说明**:
> - `-DskipTests`: 跳过测试，加快编译速度

### 步骤 3：启动 WMS 应用

编译成功后，执行以下命令启动应用：

```bash
java -jar wms-web/target/wms-web-1.0.0-SNAPSHOT.jar --spring.profiles.active=dev
```

### 步骤 4：验证启动成功

启动成功后，控制台会显示类似如下信息：

```
2026-05-16 12:50:46.815  INFO 13816 --- [           main] com.jd.wms.web.WmsApplication            : Started WmsApplication in 7.408 seconds (JVM running for 7.881)
```

## 八、常见问题

### Q1: Redis 连接失败
**原因**: Redis 服务未启动或端口被占用
**解决方案**: 
1. 确认 Redis 窗口正在运行
2. 检查端口 6379 是否被其他程序占用

### Q2: Maven 编译失败
**原因**: 依赖下载失败或代码错误
**解决方案**:
1. 检查网络连接
2. 执行 `mvn clean install -U` 更新依赖

### Q3: 数据库连接失败
**原因**: MySQL 服务未启动或配置错误
**解决方案**:
1. 确认 MySQL 服务正在运行
2. 检查 `application-dev.yml` 中的数据库配置
